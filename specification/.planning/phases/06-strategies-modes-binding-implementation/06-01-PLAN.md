<objective>
Implement the Hoist entry strategy's `.s()` post-processing and wire EntryPolicy into the segment grouping pipeline.

Purpose: The Hoist strategy is the most complex entry strategy behavior — it converts `inlinedQrl()` calls into `_noopQrl()` const declarations + `.s()` registration calls. This is required for the optimizer to be a drop-in SWC replacement. The other 6 strategies already have EntryPolicy implementations but they are not yet wired into the segment emission path.

Output: Working Hoist `.s()` post-processing in transform.rs, EntryPolicy integration in lib.rs, all 7 strategies producing correct output.
</objective>

  <action>
    **Part A: Hoist .s() post-processing in transform.rs (per D-40)**

    Add `hoist_qrl_to_module_scope()` method to QwikTransform. This should be called right after `create_inline_qrl()` when `entry_strategy == EntryStrategy::Hoist`. The method:

    1. **Early return for Lib mode** — return the call expression unchanged (Rule 7 from spec)
    2. **Extract fn_body** — remove the first argument from inlinedQrl args (the closure body)
    3. **Extract capture array** — check if last arg is an array literal, extract and remove if so
    4. **Get symbol name** — from the string literal argument (second arg)
    5. **Build _noopQrl call** — replace `inlinedQrl` callee with `_noopQrl` (or `_noopQrlDEV` in dev modes), keep remaining args (symbol name, dev info). Add PURE annotation.
    6. **Create const declaration** — `const q_{symbol_name} = /*#__PURE__*/ _noopQrl(...)` — push to `self.extra_top_items` (deduplicated by symbol_name key)
    7. **Build .s(fn_body) call** — For global identifiers, push to a new `self.ref_assignments: Vec<String>` field. For non-global identifiers, produce comma expression `(q_X.s(value), q_X)`.
    8. **Append .w([captures])** — if capture array exists, append `.w(captures)` to the result expression
    9. **Return** — identifier reference (global) or comma expression (non-global)

    Add `ref_assignments: Vec<String>` field to QwikTransform. These get emitted in `exit_program` AFTER extra_top_items const decls but BEFORE export statements (critical ordering per Pitfall 1).

    Update `exit_program` in the Traverse impl to drain `ref_assignments` as expression statements after the `extra_top_items` const declarations.

    The global check uses `self.global_collect().root.contains_key(&ident_name)` — root-level declarations are globally accessible; function params and locals are not.

    **Part B: Wire EntryPolicy into segment grouping in lib.rs**

    Currently lib.rs does not use EntryPolicy for segment grouping — segments are emitted individually. The EntryPolicy is only used indirectly via `is_inline()` checks. For the full entry strategy support, the `entry` field on SegmentRecord (already set during transform) controls grouping. Verify that `parse_entry_strategy` is used to set the `entry` field on each SegmentRecord during the transform pass. If not already wired, add the policy call in QwikTransform where SegmentRecords are created (the `entry` field should come from `policy.get_entry_for_sym()`).

    Check if QwikTransform already uses EntryPolicy to populate `SegmentRecord.entry` — if yes, Part B is already done and this is just verification. If not, wire it in.

    Do NOT touch NAPI/WASM bindings or emit mode behavior — those are in separate plans.
