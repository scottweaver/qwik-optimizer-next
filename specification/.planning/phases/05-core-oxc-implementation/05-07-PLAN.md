    **code_move.rs -- Segment module construction (CONV-05):**

    Build segment modules as strings from the data collected during traversal. Per research Pattern 2, extract segment function bodies by slicing the original source code at recorded spans -- do NOT clone AST nodes.

    The 13-step pipeline (from Jack's implementation, validated against spec):
    1. Collect the segment's body source code (from span recorded during traversal)
    2. Determine needed imports (from capture analysis categories 1 and 2)
    3. Build self-import statements for module-level declarations referenced by this segment
    4. Build re-emitted import statements for user imports referenced by this segment
    5. Build the export declaration: `export const segName = body;`
    6. Handle migrated variables (from dependency_analysis) -- prepend to segment
    7. Handle nested segments (parent-child relationships)
    8. Assemble the complete segment module as a string
    9. Re-parse with OXC Parser for normalization
    10. Run Codegen to produce final output code
    11. Generate source map if enabled
    12. Build SegmentAnalysis metadata (origin, name, hash, display_name, etc.)
    13. Return TransformModule with code, map, and segment analysis

    Key function:

    **emit.rs -- Codegen wrapper:**
    ```rust
    pub(crate) fn emit_code(
        source: &str,
        filename: &str,
        source_maps: bool,
    ) -> (String, Option<String>) {
        let allocator = Allocator::default();
        let ret = Parser::new(&allocator, source, SourceType::mjs()).parse();
        if ret.panicked {
            return (source.to_string(), None);
        }
        let mut codegen = Codegen::new();
        if source_maps {
            codegen = codegen.with_source_text(source);
        }
        let result = codegen.build(&ret.program);
        (result.code, result.map.map(|m| m.to_json_string()))
    }
    ```

    **dependency_analysis.rs -- Variable migration (CONV-05 sub-feature):**
    - Analyze which root-level declarations are exclusively referenced by a single segment
    - If a root `const`/`function`/`class` is used ONLY by one segment and NOT by the root module or any other segment: migrate it INTO that segment module
    - Remove it from the root module output
    - This reduces root module size and makes segments more self-contained

    **clean_side_effects.rs -- Treeshaker cleanup (CONV-09 support):**

    ```rust
    pub fn transform_modules(options: TransformModulesOptions) -> TransformOutput {
        let mut output = TransformOutput::default();

        let process_file = |input: &TransformModuleInput| -> TransformOutput {
            transform_code(TransformCodeOptions {
                path: &input.path,
                dev_path: input.dev_path.as_deref(),
                code: &input.code,
                src_dir: &options.src_dir,
                root_dir: options.root_dir.as_deref(),
                source_maps: options.source_maps,
                minify: options.minify.clone(),
                transpile_ts: options.transpile_ts,
                transpile_jsx: options.transpile_jsx,
                preserve_filenames: options.preserve_filenames,
                explicit_extensions: options.explicit_extensions,
                entry_strategy: options.entry_strategy.clone(),
                mode: options.mode.clone(),
                scope: options.scope.as_deref(),
                core_module: options.core_module.as_deref(),
                strip_exports: options.strip_exports.as_deref(),
                strip_ctx_name: options.strip_ctx_name.as_deref(),
                strip_event_handlers: options.strip_event_handlers,
                reg_ctx_name: options.reg_ctx_name.as_deref(),
                is_server: options.is_server,
            })
        };

        #[cfg(feature = "parallel")]
        {
            use rayon::prelude::*;
            let results: Vec<_> = options.input.par_iter().map(process_file).collect();
            for mut r in results {
                output.append(&mut r);
            }
        }

        #[cfg(not(feature = "parallel"))]
        {
            for input in &options.input {
                let mut r = process_file(input);
                output.append(&mut r);
            }
        }

        output
    }
    ```
