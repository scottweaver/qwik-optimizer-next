  <action>
    Build `tests/snapshot_tests.rs` -- a custom test harness that:

    1. **Fixture config types** -- Define serde-compatible types matching fixtures.json:
       ```rust
       #[derive(Deserialize)]
       struct FixtureFile { version: u32, fixtures: HashMap<String, FixtureConfig> }
       #[derive(Deserialize)]
       struct FixtureConfig {
           src_dir: String,
           root_dir: Option<String>,
           source_maps: bool,
           minify: String,
           transpile_ts: bool,
           transpile_jsx: bool,
           preserve_filenames: bool,
           explicit_extensions: bool,
           entry_strategy: String,
           mode: String,
           scope: Option<String>,
           core_module: Option<String>,
           strip_exports: Option<Vec<String>>,
           strip_ctx_name: Option<Vec<String>>,
           strip_event_handlers: bool,
           reg_ctx_name: Option<Vec<String>>,
           is_server: Option<bool>,
           inputs: Vec<FixtureInput>,
       }
       #[derive(Deserialize)]
       struct FixtureInput { path: String, dev_path: Option<String>, code: String }
       ```

    2. **Snapshot parser** -- Parse the custom `.snap` format into structured data:
       - Header (between `---` markers): source, assertion_line, expression
       - `==INPUT==` section: input source code
       - Segment sections: `===== filename (ENTRY POINT)==` with code, source map (`Some("...")`), and SegmentAnalysis JSON (between `/*` and `*/`)
       - Root module section: `===== filename ==` with code and source map
       - `== DIAGNOSTICS ==` section: JSON array

       Parse into a `SnapshotData` struct:
       ```rust
       struct SnapshotData {
           input: String,
           segments: Vec<SegmentSnapshot>,
           root_module: Option<RootSnapshot>,
           diagnostics: String,
       }
       struct SegmentSnapshot {
           filename: String,
           is_entry: bool,
           code: String,
           source_map: Option<String>,
           analysis_json: String,
       }
       struct RootSnapshot {
           filename: String,
           code: String,
           source_map: Option<String>,
       }
       ```

    3. **Test generation** -- Use a macro or `#[test]` functions to generate one test per fixture. For now, each test should:
       - Load fixture config from fixtures.json
       - Parse the corresponding .snap file
       - Convert fixture config to `TransformModulesOptions`
       - Call `transform_modules()` (will fail until transform is implemented)
       - Compare output against parsed snapshot

       For the initial plan, mark tests as `#[ignore]` with a comment explaining they'll be un-ignored as transforms are implemented. Include a few smoke tests that just verify the snapshot parser works correctly (parse a known .snap file, assert sections are extracted).

    4. **Snapshot comparison logic** -- Compare `TransformOutput` against `SnapshotData`:
       - Match segments by canonical_filename
       - Compare segment code (trim whitespace for cosmetic tolerance)
       - Compare SegmentAnalysis fields (JSON structural comparison)
       - Compare root module code
       - Compare diagnostics

    5. **Harness must compile** even though `transform_modules` doesn't exist yet. Use conditional compilation or a stub function that returns empty TransformOutput. The snapshot parsing and comparison code must be fully implemented.
  </action>
