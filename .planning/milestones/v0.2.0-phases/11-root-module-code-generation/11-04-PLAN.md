**Category A: Unreferenced variable declarations (per Pitfall 5 from research).**
SWC strips `const name = wrapper(q_...)` to just `wrapper(q_...)` when `name` is not referenced elsewhere in the root module. This happens through SWC's DCE `simplify::simplifier`.

Fix approach: After the fixpoint loop in Step 9, add a pass that converts unreferenced const declarations with wrapper call initializers to bare expression statements:

```rust
// After fixpoint loop, strip unreferenced wrapper call declarations
// e.g., `const renderHeader2 = component(q_...)` -> `component(q_...)`
// Only for declarations where:
//   1. The variable name is NOT referenced elsewhere in root
//   2. The initializer is a call expression (wrapper call)
//   3. The declaration is NOT exported

// Collect all referenced names (excluding declarations themselves)
let referenced: HashSet<String> = { /* collect from non-decl statements */ };

for stmt in program.body.iter_mut() {
    if let Statement::VariableDeclaration(decl) = stmt {
        if decl.declarations.len() == 1 {
            let d = &decl.declarations[0];
            if let BindingPattern::BindingIdentifier(id) = &d.id {
                if !referenced.contains(id.name.as_str()) && d.init.is_some() {
                    // Check if init is a call expression
                    if matches!(d.init.as_ref().unwrap(), Expression::CallExpression(_)) {
                        // Convert to expression statement
                        // Replace: const name = call(args) -> call(args);
                    }
                }
            }
        }
    }
}
```

Implementation note: Since OXC AST nodes are arena-allocated and we can't easily mutate Statement variants in-place, the cleanest approach may be to collect indices of statements to transform, then rebuild them. Alternatively, re-parse the expression statement from the init expression's codegen output.

**Category C: Comment separators (D-09).**
SWC emits `//` separator comments between:

Check `emit.rs` for existing `insert_separator_comments` logic. If separators are missing or in wrong positions, adjust the exit_program to insert empty statement comments or newlines at the correct positions.

**Category D: Deduplication.**
