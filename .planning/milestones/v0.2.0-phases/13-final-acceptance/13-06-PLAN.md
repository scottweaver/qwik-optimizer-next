The SWC Inline strategy emits a specific pattern:
```javascript
const q_Child_component_HASH = /*#__PURE__*/ _noopQrl("Child_component_HASH");
q_Child_component_HASH.s(()=>{ /* function body */ });
q_Child_component_HASH.w(["captured1", "captured2"]);
```

Where:
- `_noopQrl("name")` creates a placeholder QRL
- `.s(() => { body })` sets the synchronous function body
- `.w(["captures"])` sets the captured variables (only when captures exist)

The OXC implementation currently emits something fundamentally different (direct inline). Rework the Inline strategy code generation to match SWC's pattern. This is a significant change to `entry_strategy.rs`.

Key points:
1. For each segment with Inline strategy, emit `const q_NAME = _noopQrl("name")`
2. Then emit `q_NAME.s(() => { body })`
3. Then emit `q_NAME.w(["captures"])` if captures exist
4. Set `needs_noop_qrl_import = true` (already exists)
5. The body should be the original function body, not the segment-extracted code

Compare with SWC reference implementation line-by-line to ensure format matches.

Run full test suite after changes.
