<objective>
Implement JSX attribute segment extraction so that `$`-suffixed JSX attributes (e.g., `onClick$={() => {...}}`) produce separate SegmentRecord entries, matching SWC's behavior.

Purpose: This is the root cause of ~76 segment count mismatches. The current OXC optimizer only extracts segments from explicit `$()` call expressions but misses dollar-suffixed JSX attribute values entirely.

Output: Updated `transform.rs` with `jsx_event_to_html_attribute` function and JSX attribute segment extraction integrated into the `exit_expression` JSX path. Updated `jsx_transform.rs` to surface dollar-suffixed attributes for extraction.
</objective>

```rust
/// Convert a `$`-suffixed JSX event attribute name to an HTML attribute name.
///
/// Returns `Some(html_attr)` for event handler attributes, `None` for non-event props.
///
/// Examples:
/// - `onClick$` -> `Some("q-e:click")`
/// - `onDblClick$` -> `Some("q-e:dbl-click")`
/// - `document:onFocus$` -> `Some("q-d:focus")`
/// - `window:onClick$` -> `Some("q-w:click")`
/// - `onDOMContentLoaded$` -> `Some("q-e:d-o-m-content-loaded")`
/// - `render$` -> `None` (not an event handler)
fn jsx_event_to_html_attribute(jsx_event: &str) -> Option<String>
```

Implementation rules (directly from SWC lines 4658-4710):
1. Return `None` if name does not end with `$`
2. Determine prefix and index: `window:on` -> `("q-w:", 9)`, `document:on` -> `("q-d:", 11)`, `on` -> `("q-e:", 2)`, else return `None`
3. Extract name = `jsx_event[idx..len-1]` (strip trailing `$`)
4. Special case: if name is `"DOMContentLoaded"` -> return `format!("{prefix}-d-o-m-content-loaded")`
5. If name starts with `-`, strip the leading hyphen (marker for case-sensitive name)
6. Otherwise, lowercase the name
7. Apply camelCase -> kebab-case: for each char in the processed name, if uppercase or `-`, emit `-` then lowercase char; else emit char as-is

Also add the two helper functions `get_event_scope_data_from_jsx_event(jsx_event: &str) -> (&str, usize)` and `create_event_name(name: &str, prefix: &str) -> String` matching SWC's implementation exactly.
