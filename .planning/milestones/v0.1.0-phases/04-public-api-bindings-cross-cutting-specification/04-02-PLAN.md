Write 6 migration pattern subsections from the research catalog:

### Migration Pattern 1: Fold/VisitMut to Traverse Trait
- SWC pattern: `impl Fold for QwikTransform` with `fn fold_call_expr()` taking ownership
- OXC pattern: `impl Traverse<'a> for QwikTransform` with `fn enter_call_expression(&mut self, node: &mut CallExpression<'a>, ctx: &mut TraverseCtx<'a>)`
- Key differences: enter/exit hooks, TraverseCtx parameter, in-place mutation via `*node = replacement`
- Code example from Scott's transform.rs showing the Traverse impl

### Migration Pattern 2: SyntaxContext to Scoping/SymbolId
- SWC: `Id` type is `(Atom, SyntaxContext)`, identity from resolver marks
- OXC: `SemanticBuilder` produces `Scoping` with `SymbolId`/`ReferenceId`
- Key difference: identity is a side table, not a property of identifier nodes
- Note: after AST mutation, semantic info may become stale for mutated regions

### Migration Pattern 3: Ownership Transfer to Arena Allocation
- SWC: `Box<Expr>` heap-allocated, `std::mem::replace` for swaps
- OXC: `Box<'a, Expression<'a>>` arena-allocated, new nodes via `AstBuilder` or `TraverseCtx::ast`
- Key constraint: cannot clone nodes across arenas

### Migration Pattern 4: Code Move from AST Cloning to String Construction
- SWC: `code_move` module clones expression nodes into new Module nodes
- OXC: Extract segment body as source string (span slicing), build segment from string concatenation
- Why: arena allocation makes cross-arena AST construction impossible
- This is arguably simpler than the SWC approach

### Migration Pattern 5: GlobalCollect to SemanticBuilder + Collector
- SWC: manual `GlobalCollect` visitor catalogs imports/exports/declarations
- OXC: `SemanticBuilder` provides scope/symbol tables, separate `collector::collect()` for dollar-specific metadata
- Key difference: OXC gives scope resolution "for free" but needs a supplemental pass for $-semantics

### Migration Pattern 6: Deferred Statement Insertion via exit_program
- SWC: `fold_module` directly modifies module body (ownership model allows it)
- OXC: Cannot insert siblings during enter hooks. Accumulate in Vec, insert all in `exit_program`
- Pattern: pending_imports Vec + pending_hoisted Vec, flushed at exit
