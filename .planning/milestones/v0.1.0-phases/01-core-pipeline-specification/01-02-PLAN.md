Add a "### Dollar Detection (CONV-01)" section under "## Stage 4: Core Transform" in the spec document. Structure per D-04:

**Behavioral Rules:**
1. **Imported markers**: Any named import from `@qwik.dev/core` whose specifier ends with `$` (the QRL_SUFFIX constant) is a marker function. Detected during QwikTransform construction by iterating `global_collect.imports` and checking `specifier.ends_with('$')`. The marker set is stored in a `marker_functions: HashMap<Id, JsWord>` mapping local Id to the original specifier name.
2. **Local markers**: Any locally-defined export whose name ends with `$` is also a marker function. Detected via `global_collect.export_local_ids()` with `id.0.ends_with('$')`. These are added to the same `marker_functions` HashMap.
3. **Callee conversion rule**: When a `$`-suffixed call is detected, the callee is replaced with the `Qrl` variant. `convert_qrl_word()` (transform.rs:179-187) replaces the trailing `$` with `Qrl` — e.g., `component$` -> `componentQrl`, `useTask$` -> `useTaskQrl`. The bare `$` function maps to `qrl` (the QSEGMENT case).
4. **Special handling**: `sync$` has its own code path (serialization, not segment extraction). The generic `$()` maps to QSEGMENT. `component$` additionally triggers PURE annotation on its `componentQrl` wrapper.
5. **Detection site**: Occurs in `visit_mut_call_expr` — when a CallExpr's callee is an Identifier found in `marker_functions`, the dollar detection triggers.
6. **Non-markers**: Functions that happen to end with `$` but are NOT imported from `@qwik.dev/core` and NOT locally exported are NOT markers. They pass through unchanged.

**Source reference:** transform.rs:189-202 (marker construction), words.rs (QRL_SUFFIX = '$'), transform.rs:179-187 (convert_qrl_word)

**Examples per D-14 (3 examples: basic, import variation, edge case):**

2. "Multiple Markers from Same Import (example_qrl_usage or similar)": Show `component$`, `useTask$`, `useVisibleTask$` all detected from the same `@qwik.dev/core` import
3. "Non-Marker Dollar Function": Show a locally-defined function `myHelper$()` that is NOT exported — it should NOT trigger detection. Contrast with an exported `myHelper$()` that IS detected.

Name examples per D-16 with Jack's snapshot names in parentheses where applicable.

Add a "### QRL Wrapping (CONV-02)" section immediately after Dollar Detection. Structure per D-04:

**Behavioral Rules:**
1. **Three QRL creation paths** — which path is taken depends on entry strategy and whether the segment is stripped:
   - `create_qrl()` (transform.rs:1888): For Segment/Hook/Single/Component/Smart strategies. Produces:
     ```javascript
     qrl(() => import("./segment_path"), "symbol_name")
     // or with captures:
     qrl(() => import("./segment_path"), "symbol_name", [capture1, capture2])
     ```
   - `create_inline_qrl()` (transform.rs:1945): For Inline/Hoist strategies. Produces:
     ```javascript
     inlinedQrl(original_fn_expr, "symbol_name")
     // or with captures:
     inlinedQrl(original_fn_expr, "symbol_name", [capture1, capture2])
     ```
   - `create_noop_qrl()` (transform.rs:3000): When `should_emit_segment()` returns false (callback stripped via `strip_ctx_name` or `strip_event_handlers`). Produces:
     ```javascript
     _noopQrl("symbol_name")
     ```

2. **Dev mode variants**: When emit mode is Dev or Hmr:
   - `qrl` -> `qrlDEV` with extra source location arg: `{file: "path", lo: start_offset, hi: end_offset, displayName: "name"}`
   - `inlinedQrl` -> `inlinedQrlDEV` with same extra arg
   - `_noopQrl` -> `_noopQrlDEV`

3. **Captures emission** (transform.rs:2013-2029): When `scoped_idents` (from capture analysis) is non-empty, an ArrayLiteral of captured variable identifiers is appended as the last argument. Additionally, a `.w([captures])` method call is added to the QRL reference for the "with captures" registration.

4. **PURE annotation**: Only `componentQrl(...)` calls get `/*#__PURE__*/` annotation. All other `*Qrl` wrappers (useTaskQrl, useVisibleTaskQrl, etc.) do NOT get PURE because they have side effects. This is CONV-08 behavior but triggered during QRL wrapping.

5. **Symbol name**: The second argument is always the symbol name string (e.g., `"sym1_aXUrPXX5Lak"`). This is the hash-suffixed name from the Hash Generation algorithm. It serves as the segment identifier for lazy loading.

6. **Import path**: The first argument to `qrl()` is a dynamic import arrow function: `() => import("./canonical_filename")`. The path is constructed per the Path Resolution algorithm.

**Source reference:** transform.rs:1888-2062 (create_qrl, create_inline_qrl), transform.rs:3000 (create_noop_qrl), transform.rs:2013-2029 (captures)

**Examples per D-14 (3 examples: basic, with captures, dev mode):**
1. "Basic QRL Wrapping (example_6)": Show `$((ctx) => ...)` becoming `qrl(() => import("./path"), "symbol_name")` in root, with segment exporting the function

3. "Dev Mode QRL (example_dev_mode or constructed)": Show the same transformation producing `qrlDEV` with `{file, lo, hi, displayName}` metadata object
