Add two sections to the Infrastructure area of the spec document:

**Hash Generation section (SPEC-23):**
- Two-level hashing algorithm:
  1. **File hash**: `DefaultHasher` seeded with optional `scope` string + `rel_path` (forward-slashed). Produces a u64 used as prefix for JSX key generation. Source: transform.rs:204-210.
  2. **Symbol hash**: `DefaultHasher` seeded with (in order): `scope` (if set, written first via Hasher::write), then `rel_path` (forward-slashed), then `display_name` (escaped, deduplicated). Source: transform.rs:393-421.
- Base64 encoding algorithm (exact steps from transform.rs:4725):
  1. Take u64 hash value
  2. Convert to little-endian bytes: `u64.to_le_bytes()`
  3. Encode with `base64::engine::general_purpose::URL_SAFE_NO_PAD`
  4. Replace all `-` and `_` characters with `0`
- Symbol name format:
  - Dev/Test/Hmr/Lib mode: `{display_name}_{hash}` (e.g., `sym1_aXUrPXX5Lak`)
  - Prod mode: `s_{hash}` (e.g., `s_aXUrPXX5Lak`)
- Display name construction algorithm (from transform.rs:4612 escape_sym + stack_ctxt):
  1. `stack_ctxt.join("_")` builds context hierarchy
  2. `escape_sym()`: replace non-alphanumeric with `_`, trim/squash consecutive underscores
  3. Prepend `_` if starts with digit
  4. Deduplication via `segment_names` HashMap: first occurrence gets base name, subsequent get `_N` suffix (N = count)
  5. Final: `{file_stem}_{display_name}` (e.g., `test.tsx_sym1`)
- Examples per D-14:
  1. Basic: Simple `$()` call showing display_name construction + hash computation + final symbol name
  2. Deduplication: Two `$()` calls in same function showing `_1` suffix on the second

**Path Resolution section (SPEC-24):**
- `parse_path()` behavior (parse.rs:207-210): extracts file stem, extension, determines is_typescript/is_jsx from extension
- `get_canonical_filename()` (transform.rs:4910-4912): `{display_name}_{hash_suffix}` where hash_suffix is the last `_`-delimited token of symbol_name
- Import path construction: `"./" + canonical_filename` (with extension appended if `explicit_extensions` config is true)
- Extension mapping: input `.tsx` -> output `.tsx`, input `.ts` -> `.js`, input `.jsx` -> `.js`, input `.js` -> `.js`
- Source references per D-06
- Examples per D-14:
  1. Basic: Input path `src/components/Counter.tsx` -> canonical filename for a segment
  2. Edge case: `explicit_extensions: true` vs `false` showing import path difference
