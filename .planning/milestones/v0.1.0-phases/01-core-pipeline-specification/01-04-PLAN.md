Add a "### Segment Extraction (CONV-05)" section after QRL Wrapping in the spec. Structure per D-04:

**Behavioral Rules:**

1. **Segment creation** (`create_segment` in transform.rs:1110-1145):
   - Computes `canonical_filename` from display_name + symbol_name (using Hash Generation and Path Resolution algorithms documented in Plan 01)
   - Gets `entry` classification from `entry_policy` (determines bundler chunking — documented in Phase 3)
   - Builds `import_path`: `"./" + canonical_filename` (appends extension if `explicit_extensions` config is true)
   - Creates the QRL call via `create_qrl()` (documented in QRL Wrapping section)
   - Pushes a `Segment` struct to `self.segments` vector for later processing

2. **Segment module construction** (`new_module` in code_move.rs:122+):
   Seven-step process to build a complete JS module from a segment:
   a. **Captures injection**: If segment has `scoped_idents` (non-empty captures from Capture Analysis), add `import { _captures } from "@qwik.dev/core"` and inject `const {name} = _captures[N]` destructuring statements at the top of the function body. Each captured variable gets an index-based access.
   b. **Import resolution for local_idents**: For each identifier used by the segment body, check GlobalCollect:
      - If it's a user import -> re-emit the import statement with same source (e.g., `import css from './style.css'`)
      - If it's an export but not an import -> generate self-import: `import { _auto_X as X } from "./module_stem"`
      - Collision detection: if same local name needed for different sources, rename with `_N` suffix
   c. **Extra top items**: Add hoisted QRL declarations needed by this segment (e.g., if segment contains a nested `$()`, the inner QRL reference is hoisted)
   d. **Migrated root vars**: Add variable declarations moved from root module (if variable migration applies — see Variable Migration section)
   e. **Topological sort**: Order all items by dependency to ensure declarations appear before their uses
   f. **Deduplication**: Remove duplicate items by symbol name
   g. **Named export**: Create the final export: `export const {symbol_name} = {function_expression}`

3. **Nested segments** (segment_stack):
   - `segment_stack` tracks nesting depth during AST traversal
   - When a `$()` is encountered inside another `$()` body, `parent_segment` is set to `segment_stack.last()`
   - The inner segment's `Segment.parent` field points to the outer segment
   - A component's nested segments (event handlers, useTask$, etc.) have `parent` pointing to the component segment
   - This parent-child relationship is exposed in `SegmentAnalysis.parent` field

4. **Segment struct fields** (what gets recorded):
   - `name`: symbol name (hash-suffixed)
   - `canonical_filename`: for the output module path
   - `data`: the extracted function expression (AST node)
   - `scoped_idents`: captured variables (from Capture Analysis)
   - `local_idents`: all identifiers referenced by the segment body
   - `entry`: entry classification from entry_policy
   - `parent`: optional parent segment name
   - `ctx_kind`: SegmentKind (Function, EventHandler, JSXProp)
   - `ctx_name`: the original marker name (e.g., "component$", "useTask$")

**Source reference:** transform.rs:1110-1145 (create_segment), code_move.rs:105-450 (new_module), parse.rs:446-583 (segment construction loop)

Add a "### Import Rewriting (CONV-12)" section after Segment Extraction. Structure per D-04:

**Behavioral Rules — Four mechanisms:**

1. **Legacy rename** (Step 5 of pipeline, runs early):
   - `RenameTransform` (rename_imports.rs) rewrites `@builder.io/qwik` to `@qwik.dev/core`
   - Also handles `@builder.io/qwik/build` -> `@qwik.dev/core/build`
   - Runs BEFORE GlobalCollect so all subsequent analysis sees normalized paths
   - Source: rename_imports.rs, parse.rs:278

2. **Consumed import stripping** (automatic via DCE):
   - After QwikTransform, imports for `$`-suffixed functions (e.g., `component$`, `$`) are no longer referenced in the root module (the callee was replaced with `componentQrl`, `qrl`, etc.)
   - SWC's DCE/hygiene pass removes unreferenced imports automatically
   - The optimizer does NOT explicitly strip these — it relies on post-transform cleanup
   - Example: `import { $, component$ } from '@qwik.dev/core'` -> both `$` and `component$` are consumed, only `qrl` and `componentQrl` remain

3. **Synthetic import addition** (during transform):
   - `ensure_core_import(specifier)` registers new imports needed by transformed code
   - Added to GlobalCollect's imports with `synthetic: true` flag
   - Common synthetic imports: `qrl`, `componentQrl`, `inlinedQrl`, `_jsxSorted`, `_jsxSplit`, `_fnSignal`, `_captures`, `_noopQrl`
   - These appear in the root module's import statement after codegen
   - Source: transform.rs ensure_core_import calls throughout

4. **Per-segment import resolution** (code_move.rs:200-276):
   For each identifier used by a segment, `resolve_import_for_id` checks:
   a. **User import (exact match)**: GlobalCollect has an import with matching Id -> re-emit with same source module. E.g., `import { foo } from './utils'` in original -> same import in segment.
   b. **User import (unique-by-symbol fallback)**: If exact Id match fails but there's a unique import with the same symbol name -> use it (handles SyntaxContext mismatches).
   c. **Export (not import)**: Identifier is in GlobalCollect exports but not imports -> generate self-import: `import { _auto_X as X } from "./module_stem"`. The `_auto_` prefix is a synthetic export name added by `ensure_export()`.
   d. **Collision handling**: If the same local name is needed for two different import sources, the second gets renamed with a `_N` suffix to avoid conflicts.
   e. **Runtime imports**: `_captures` is imported from `@qwik.dev/core` when the segment has captured variables. Other runtime helpers (`qrl`, etc.) may appear if the segment contains nested `$()` calls.

**Source reference:** rename_imports.rs (legacy rename), code_move.rs:66-277 (per-segment resolution), collector.rs ImportKind + import() method

1. "Root Module Import Transformation (example_6)": Show before (user imports `$`, `component$`) and after (synthetic imports `qrl`, `componentQrl`; original `$` and `component$` consumed/stripped)
