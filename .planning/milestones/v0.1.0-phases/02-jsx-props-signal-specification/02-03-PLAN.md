Structure the section as follows (per D-22, both _wrapProp and _fnSignal in same section):

**#### Decision Flow (create_synthetic_qqsegment)**

Document the gateway function's decision flow as numbered steps:
1. Collect all identifiers used in the expression (IdentCollector)
2. Partition declaration stack into scope variables (Var) and invalid declarations
3. Check disqualifying conditions:
   - Any identifier is a global (import/external) -> contains_side_effect = true
   - Any identifier is an invalid declaration (function, class) -> bail out, return no wrapping
   - Any identifier not in declaration stack at all -> bail out, return no wrapping
4. Compute scoped identifiers and const-ness
5. If contains_side_effect -> bail out, no wrapping
6. If expression is simple Ident -> no wrapping needed (just the signal reference, not .value)
7. If expression is Call or Tpl (template literal) and not const -> bail out, no wrapping
8. If member expression on ident (e.g., signal.value) -> _wrapProp(signal)
9. Everything else -> attempt convert_inlined_fn for _fnSignal

**#### _wrapProp: Signal Property Access (per D-22)**

Document both forms:
- **One-argument form:** _wrapProp(signal) — generated when expression is ident.prop (member expression where object is simple identifier). Wraps the signal object; runtime knows to access .value. Works for signal.value, store.address, etc.
- **Two-argument form:** _wrapProp(_rawProps, "propName") — generated when props destructuring (CONV-04) has transformed a destructured prop to _rawProps.propName. Signal optimization sees this as member access on _rawProps. Cross-reference Props Destructuring (CONV-04) section.

Source: transform.rs:791-804

**#### _fnSignal: Computed Expression Wrapping**

Document convert_inlined_fn (inlined_fn.rs:24-115):

Bail-out conditions (returns None, no wrapping):
1. Expression is ArrowExpr
2. Expression contains a CallExpr node (mark as non-const)
3. Expression identifiers are NOT used as objects (no member access on them)
4. ReplaceIdentifiers visitor encounters: nested ArrowExpr, Function, ClassExpr, Decorator, Stmt, or disallowed Callee
5. Rendered expression exceeds 150 characters (document this explicitly per Pitfall 4 — behavioral consequence is loss of fine-grained reactivity tracking, not an error)
6. scoped_idents is empty (all const) -> no wrapping needed, expression is const

Successful generation:
1. Create positional parameter names: p0, p1, p2, ... (one per scoped identifier)
2. Replace all captured identifier references with positional params
3. Render transformed expression to minified string (the "stringified version")
4. Wrap in arrow function: (p0, p1) => expression_with_p0_p1
5. Generate call: _fnSignal(arrowFn, [capturedVar0, capturedVar1], "stringified")

Arguments:
1. Arrow function with positional params
2. Array of captured variable references (actual runtime values)
3. (Optional, server mode) Stringified version for SSR

**#### _fnSignal Hoisting**

Document hoist_fn_signal_call (transform.rs:2763-2872):
1. Arrow function (1st arg) hoisted to module-level const: const _hf0 = (p0) => 12 + p0.value;
2. Stringified expression (3rd arg) hoisted to module-level const: const _hf0_str = "12+p0.value";
3. _fnSignal call rewritten to reference hoisted names: _fnSignal(_hf0, [signal], _hf0_str)
4. Counter-based naming: _hf0, _hf1, _hf2, etc.
5. Deduplication: same arrow function body reuses existing hoisted reference

**#### Application Boundaries Decision Table (per D-21)**

Create an explicit table covering ALL expression type x context combinations:

| Expression | Captured Vars | Side Effects | Result | Example |
|-----------|--------------|--------------|--------|---------|
| Simple identifier | any | any | No wrapping | signal -> signal |
| ident.prop (member on ident) | yes | no | _wrapProp(ident) | signal.value -> _wrapProp(signal) |
| ident.prop (member on ident) | yes | yes (global) | No wrapping | globalThing.thing -> as-is |
| Computed expression | yes, non-empty | no | _fnSignal(fn, [...]) | 12 + signal.value -> _fnSignal(_hf0, [signal], _hf0_str) |
| Computed expression | empty (all const) | no | No wrapping, const | 1 + 2 -> 1 + 2 |
| Computed expression | yes | yes (global) | No wrapping | signal.value + globalFn() -> as-is |
| Arrow/function expression | any | any | No wrapping (bail) | () => {} -> as-is |
| Call expression | any | any | No wrapping, non-const | fn() -> fn() |
| Template literal (non-const) | yes | no | No wrapping, non-const | `${signal.value}` -> as-is |
| Expression > 150 chars | yes | no | No wrapping, non-const | (long) -> as-is |
| Ternary/binary | yes, non-empty | no | _fnSignal(fn, [...]) | a.value ? 'yes' : 'no' -> _fnSignal(...) |
| Ternary containing call | yes | no | No wrapping (abort) | a.value ? fn() : b -> as-is |
