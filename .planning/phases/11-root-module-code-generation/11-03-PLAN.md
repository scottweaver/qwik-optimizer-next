**1. Find the SWC fold_call_expr pattern for stack_ctxt push.**

In SWC transform.rs, `fold_call_expr` does this when it detects a marker function call:
```rust
// SWC pushes the escaped ctx_name onto stack_ctxt before creating the segment
self.stack_ctxt.push(escape_sym(ctx_name));
// ... creates segment ...
self.stack_ctxt.pop();
```

Where `escape_sym` strips the `$` suffix (similar to our `escape_dollar`).

For bare `$()` calls: SWC uses `escape_sym("$")` which produces `""` (empty string after stripping $). Pushing an empty string would add a double underscore to display_name. **6. Handle the `escape_dollar` function.** It already exists in transform.rs (~line 2838):
```rust
fn escape_dollar(name: &str) -> String {
    name.strip_suffix('$').unwrap_or(name).to_string()
}
```
This is correct -- "component$" -> "component", "$" -> "".
