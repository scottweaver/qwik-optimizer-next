    **## Stage 6: QRL Special Cases**
    Brief intro: three independent behaviors that modify QRL generation or annotation.

    **### PURE Annotations (CONV-08)**
    Per D-04, D-05, D-06, D-24:
    - SWC Source: `transform.rs` — `add_pure_comment` calls, `create_internal_call` with `pure: true`
    - Pipeline Position: During Step 10 (QwikTransform pass)
    - Rules:
      1. `componentQrl(...)` gets `/*#__PURE__*/` — tree-shakeable component declaration
      2. `qrl()`, `inlinedQrl()`, `_noopQrl()` get PURE via `create_internal_call(..., pure: true)` — internal QRL factories
      3. `qSegment()` gets PURE annotation
      4. JSX calls (`_jsxSorted`, `_jsxSplit`) get PURE (already documented in Phase 2)
      5. **Anti-list:** `useTaskQrl()`, `useStyleQrl()`, `serverStuffQrl()`, and ALL other `Qrl`-suffixed wrappers do NOT get PURE — they are side-effectful registrations at component render time
    - Key distinction: declaration (componentQrl — tree-shakeable) vs. registration (useTaskQrl — side-effectful)
    - Example: use `impure_template_fns` snapshot showing componentQrl with PURE and other wrappers without
    - See also: `example_strip_exports_unused` for PURE on qrl() calls

    **### sync$ Serialization (CONV-13)**
    Per D-04, D-05, D-06, D-24:
    - SWC Source: `transform.rs` `handle_sync_qrl` + `inlined_fn.rs` `render_expr`
    - Pipeline Position: During Step 10 (detected during dollar detection, handled inline)
    - Rules:
      1. `sync$(fn)` detected when callee matches `sync_qrl_fn` identifier
      2. First argument must be ArrowExpr or FnExpr; other types passed through unchanged
      3. Function serialized to minified string via codegen with `Config::with_minify(true)`, no comments
      4. Output: `_qrlSync(originalFn, "serializedString")` — both AST expression AND string emitted
      5. NO segment extraction — unlike other `$` calls, sync$ keeps function inline
      6. Trailing semicolon stripped from serialized string; comments removed during serialization
    - Example: use `example_of_synchronous_qrl` snapshot showing three function forms (per D-16, D-13)
    - See also: list any related snapshots

    **### Noop QRL Handling (CONV-14)**
    Per D-04, D-05, D-06, D-24:
    - SWC Source: `transform.rs` `create_noop_qrl` + `hoist_qrl_to_module_scope`
    - Pipeline Position: During Step 10 (generated when should_emit_segment returns false)
    - When generated: (1) strip_ctx_name or strip_event_handlers match, (2) Hoist strategy's module-scope hoisting
    - Forms by mode:
      - Prod/Lib/Test: `_noopQrl("symbolName")`
      - Dev/Hmr: `_noopQrlDEV("symbolName", { file: "...", lo: N, hi: N, displayName: "..." })`
    - All noop QRL calls get PURE annotations (cross-reference CONV-08)
    - Noop segment output: `export const symbolName = null;` with empty source map
    - Captures preserved: even stripped segments emit `.w([captures])` call chain for runtime scope tracking
    - Example: use `example_noop_dev_mode` snapshot for Dev mode noop QRL (per D-16, D-13)
    - See also: `example_strip_server_code`, `example_inlined_entry_strategy`
