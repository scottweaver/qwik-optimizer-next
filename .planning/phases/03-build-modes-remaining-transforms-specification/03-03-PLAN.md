    **## Entry Strategies**
    Intro paragraph: EntryPolicy trait, how strategies determine segment grouping, 7 enum variants mapping to 5 implementations.

    Per D-25, each of the 5 distinct EntryPolicy implementations gets its own subsection:

    **### InlineStrategy (Inline, Hoist)**
    - EntryPolicy::get_entry_for_sym returns `Some("entry_segments")` — all segments grouped together
    - **Inline variant:** `should_inline=true`, function body inlined as first arg to `inlinedQrl(fn, "name", [captures])`
    - **Hoist variant:** `should_inline=false`, creates separate Segment, then `hoist_qrl_to_module_scope` converts to `_noopQrl("name")` + `.s(fnBody)` pattern with captures as `.w([captures])`
    - Lib mode exception: hoist returns unchanged (no hoisting in Lib mode)
    - Example: use `example_inlined_entry_strategy` snapshot showing Hoist pattern (per D-16, D-13)

    **### SingleStrategy (Single)**
    - Returns `Some("entry_segments")` — all segments in one output file
    - Unlike Inline: segments ARE extracted to separate modules, just all land in one file
    - No snapshot available — describe behavior by comparison with Inline

    **### PerSegmentStrategy (Hook, Segment)**
    - Returns `None` — each segment gets its own output file, maximum granularity
    - Hook and Segment are enum aliases for the same behavior
    - Example: use `example_strip_server_code` showing per-segment output files (per D-16, D-13)

    **### PerComponentStrategy (Component)**
    - Returns `Some("{origin}_entry_{root}")` if context stack has root component, else `Some("entry_segments")`
    - Groups segments by their enclosing component
    - Describe grouping key construction from stack_ctxt

    **### SmartStrategy (Smart)**
    - Default strategy. Conditional grouping:
      - Event handlers without captures AND (`SegmentKind::Function` or `ctx_name == "event$"`) AND `scoped_idents.is_empty()` => `None` (own file)
      - Top-level QRLs (no context stack) => `None` (own file)
      - Everything else => grouped by component like PerComponentStrategy
    - Rationale: isolates leaf event handlers for lazy-loading, groups stateful code by component

    **Summary table:** 7-row table with Strategy Enum | EntryPolicy Impl | Grouping Behavior columns (from research)
