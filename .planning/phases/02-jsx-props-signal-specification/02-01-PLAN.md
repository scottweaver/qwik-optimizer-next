3. **Behavioral Rules (numbered):**
   - Rule 1: Trigger condition — component$() calls with exactly one parameter that is an object destructuring pattern. Arrow functions with block statement or single expression body.
   - Rule 2: _rawProps replacement — Each destructured prop binding becomes _rawProps.propName member access. Renamed props ({count: c}) use original prop name: c -> _rawProps.count. Default values with const expressions use nullish coalescing: {x = 5} -> _rawProps.x ?? 5. Non-const default values cause the transform to bail out entirely.
   - Rule 3: _restProps() handling (per D-20 full behavioral detail) — When rest pattern exists ({...rest}), import _restProps from @qwik.dev/core. Insert const declaration at top of function body. Exclusion list contains original prop names (not aliases). Rest-only pattern ({...props}) omits the exclusion array argument.
   - Rule 4: Inlining optimizations — After parameter transformation, scan function body for const declarations that can be inlined: literal initializers removed and inlined at use sites, member access on known identifiers inlined, computed member access inlined. use* hook results with member access kept as separate const + member access.
   - Rule 5: Skip condition — If function body starts with _captures destructuring (pre-compiled Lib mode code), skip entire body transformation.
   - Rule 6: Shorthand property expansion — Props used in shorthand object properties ({name} in JSX) expanded to key-value: {name: _rawProps.name}.
   - Rule 7: Unused declaration cleanup — After inlining, declarations like `const _unused;` (with no initializer) are removed to avoid invalid JS output.
