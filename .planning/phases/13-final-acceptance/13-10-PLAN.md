9. Otherwise: create _fnSignal with hoisted function:
   - Serialize the expression with scope idents replaced by positional params (p0, p1, ...)
   - Look up in `hoisted_fn_signals` for dedup

Build on Task 1's infrastructure to complete the .s() body rewriting system.

**Step 1: Component body rewriting with _rawProps**

When a component$ segment has its .s() body emitted, the callback function's parameter must be `_rawProps` (not the original param name). SWC does this transformation:

Original: `component$((props)=>{ ... props.foo ... })`
Transformed .s() body: `(_rawProps)=>{ const props = _rawProps; ... }`

Or with destructuring:
Original: `component$(({color, ...rest})=>{ ... })`
Transformed: `(_rawProps)=>{ const props = _restProps(_rawProps, ["color"]); ... }`

The _rawProps rewriting rules (from SWC):
1. Simple param (not destructured): `const paramName = _rawProps;` or if only used for member access, just use `_rawProps` directly
2. Destructured with rest: `const rest = _restProps(_rawProps, ["destructured_keys"])`
3. Individual destructured props: `const propName = _rawProps.propName`

**Step 2: _captures for nested QRLs**

When a QRL inside a component body captures variables from the component scope, those become `_captures` in the NESTED segment, and `.w([captured_vars])` on the QRL ident in the component body.

Example flow:
```javascript
// Original
component$((props)=>{
    useTask$(()=>{ props.checked; });
});

// Root module (SWC output):
q_task.s(()=>{ const props = _captures[0]; props.checked; });
q_component.s((_rawProps)=>{
    useTaskQrl(q_task.w([_rawProps]));
});
```

The .w() array maps scoped_idents to their root-module-visible names. For component body props, the scoped ident `props` maps to `_rawProps` in the .s() body.

**Step 3: Non-global ident inline .s()**

SWC has special handling (lines 1561-1588) for QRL idents that are NOT global (defined inside a function, not at module scope). For these, `.s()` is emitted inline as a comma expression: `(q_X.s(value), q_X)` rather than as a separate statement.

Check if the OXC version needs this pattern. It applies when QRLs are created inside component bodies (nested dollar-sign calls).

**Step 4: Integration with existing ref_assignments**

The existing `ref_assignments` vector in transform.rs already receives `.s()` call strings. The current implementation pushes the ORIGINAL function body. Replace this with the SIGNAL-OPTIMIZED body for component segments:

1. Before emitting the .s() call, check if the segment is a component$ (ctx_name == "component$")
2. If so, build the optimized body with _rawProps rewriting and signal-optimized JSX
3. For non-component segments (useTask$, etc.), build the body with _captures destructuring

The .s() body string should contain the complete transformed callback, ready to be parsed as JavaScript.

**Step 5: .w() capture array emission**

When building the wrapper call (e.g., `componentQrl(q_name)` or `useTaskQrl(q_name)`), if the QRL has scoped_idents (captures), append `.w([capture1, capture2])`:

`useTaskQrl(q_task.w([_rawProps]))` instead of `useTaskQrl(q_task)`

The capture array contents come from the segment's scoped_idents, mapped to their root-module names (e.g., `props` -> `_rawProps` when inside a component body).
