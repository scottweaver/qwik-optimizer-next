  <action>
The SWC output adds `/*#__PURE__*/` annotations before these call expressions in the root module:
- `qrl()` / `qrlDEV()` calls in the hoisted const declarations
- `componentQrl()` calls in the body
- `_jsxSorted()` / `_jsxSplit()` calls in lightweight component bodies
- `_noopQrl()` calls (Inline strategy)

There are two approaches:
1. **AST-level**: Add `#[pure]` leading comments to the call expression AST nodes in transform.rs before codegen. OXC codegen will then emit `/* @__PURE__ */` which normalize_pure_annotations converts to `/*#__PURE__*/`.
2. **Post-processing**: After codegen, inject `/*#__PURE__*/` before specific function calls via string manipulation.

Approach 1 is preferred if OXC supports adding leading comments to expression nodes. Investigate `oxc::ast::ast::Expression` and whether it supports `leading_comments` or if `Span`-based comment injection is available.

If AST-level is not straightforward, use approach 2: add a post-processing step in emit.rs that inserts `/*#__PURE__*/` before known wrapper call patterns: `qrl(`, `qrlDEV(`, `componentQrl(`, `_jsxSorted(`, `_jsxSplit(`, `_noopQrl(`. Only insert when the annotation is not already present.

Ensure the annotation appears on the SAME LINE as the call, directly before it, separated by a space: `/*#__PURE__*/ componentQrl(...)`.
