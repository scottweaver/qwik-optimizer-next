    **Step 1: Replace the Segment strategy QRL-inline code (lines ~1492-1584) with hoisting logic:**

    Instead of modifying `call.arguments` to build `qrl(() => import("./path"), "sym")` inline, do:

    a. Build the hoisted const RHS string:
    ```rust
    let import_path = if self.explicit_extensions {
        format!("./{}.{}", names.canonical_filename, self.extension)
    } else {
        format!("./{}", names.canonical_filename)
    };
    let is_dev = matches!(self.mode, EmitMode::Dev | EmitMode::Hmr);
    let qrl_callee_name = if is_dev { "qrlDEV" } else { "qrl" };
    let ident_name = format!("q_{}", names.symbol_name);

    let qrl_rhs = format!(
        r#"/*#__PURE__*/ {}(()=>import("{}"), "{}")"#,
        qrl_callee_name, import_path, names.symbol_name
    );
    ```

    b. Push to `extra_top_items` (with deduplication, same pattern as Hoist strategy line 1331):
    ```rust
    if !self.extra_top_items.iter().any(|h| h.symbol_name == names.symbol_name) {
        self.extra_top_items.push(HoistedConst {
            name: ident_name.clone(),
            rhs_code: qrl_rhs,
            symbol_name: names.symbol_name.clone(),
        });
    }
    ```

    c. Build the replacement expression for the call site. The call site replacement depends on context:
    - If the dollar call is the ENTIRE expression (e.g., `export const x = $(() => ...)`) and there's no wrapper rename, replace with just `ident_name`
    - If the dollar call has a wrapper (e.g., `component$(() => ...)` renamed to `componentQrl()`), the call site becomes `componentQrl(q_symbolName)`
    - If there are captures, the identifier gets `.w([cap1, cap2])` appended BEFORE any wrapper

    For the call site replacement, use string-based replacement via the established pattern. Replace `call.callee` with the wrapper name and `call.arguments` with just the hoisted identifier (plus `.w([captures])` if needed).

    Specifically:
    - When `qrl_wrapper_name == pending.ctx_name.replace('$', "Qrl")` and the wrapper is NOT just "qrl" (i.e., it's `componentQrl`, `useTaskQrl`, etc.): keep the renamed callee as `qrl_wrapper_name`, set the single argument to the hoisted identifier (with `.w()` if captures)
    - When the original call was bare `$()`: replace the entire call expression with just the hoisted identifier (or `ident.w([caps])`)

    For bare `$()` calls: since we can't easily replace a CallExpression with an Identifier in OXC's AST, use the pattern of renaming the callee to an identity function or restructure to set `call.callee` to the hoisted ident and clear arguments. Actually, the simplest approach matching the established string-assembly pattern: set `call.callee` to an identifier referencing the hoisted const name, and clear `call.arguments`. But this produces `q_sym()` not `q_sym`. Instead, for bare `$()`, we need to replace the parent expression entirely.

    Alternative approach (simpler, matching SWC output exactly): For ALL strategies including Segment, instead of AST manipulation at the call site, record the replacement text and use the existing `replacements` or similar mechanism. However, looking at the current code, the simplest path is:

    For wrapper calls (componentQrl, useTaskQrl, etc.):
    - Keep `call.callee` as the renamed wrapper (already set at line 1304-1307)
    - Clear `call.arguments` and push a single argument: a parsed expression from the hoisted ident string (with `.w()` if captures exist)

    For bare `$()` calls:
    - Rename `call.callee` to the hoisted ident name
    - Clear `call.arguments`
    - If captures, this needs to be `q_sym.w([caps])` which is a MemberExpression call, not a bare identifier

    The simplest working approach: always keep it as a function call, but for bare `$()`:
    - If no captures: replace entire call with just the identifier. Since OXC AST makes this hard in a CallExpression, set callee to a wrapper that returns the value. Actually -- look at the SWC output for bare `$()`:
      ```
      export const renderHeader1 = q_renderHeader1_jMxQsjbyDss;
      ```
    This is NOT a call expression -- it's an assignment to an identifier. The call expression `$(() => ...)` was completely replaced. This means we need to replace the CallExpression AST node with an Identifier (or MemberCall for `.w()`).

    Use `ctx.ast.expression_identifier(SPAN, arena_ident(ctx, &ident_name))` to create an identifier expression, then replace the parent expression. But we're inside `exit_call_expression` which gives us `&mut CallExpression` -- we can't replace it with a non-call from here.

    Looking at the existing Hoist strategy code (lines 1388-1465), it handles this by building a replacement expression string and parsing it, then replacing call fields. Specifically it handles:
    - No captures, no wrapper: replacement = `q_sym` (but wraps in identity call to fit CallExpression shape -- check actual code)
